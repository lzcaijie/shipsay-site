# V5.1_TEMPLATE_STANDARD（Shipsay 模板标准草案）

> 目的：这份文档不是给“人看着热闹”的，而是给后续 **AI / 人工快速判断能不能改、该怎么改、哪些不能乱改** 用的。
>
> 适用范围：`shipsay-site` 分站侧模板标准，当前以 `themes/shipsay/` 作为母模板基线；其它主题（如 `88ds`）可参考，但默认不反向覆盖母模板规则。
>
> 当前状态：**v5.1 草案**。本版以“删减 v5.0 里过硬、容易误改的规则 + 补齐真实链路 + 明确验收方式”为主。

---

## 1. 单一目标

后续模板整理的主要目标只有一个：

**让 AI 或人工在看到源码后，能快速、安全地复制一套新模板，而不是为了改模板把原来正确的逻辑改坏。**

所以本标准优先级如下：

1. **不破坏现有正确逻辑**
2. **变量来源准确**
3. **模板职责清晰**
4. **页面显示正常，移动端优先**
5. **在不破坏页面协调性的前提下做统一**

---

## 2. 核心原则（必须遵守）

### 2.1 不能为了“按标准改”而把原来正确的东西改坏

这条优先级最高。

出现以下情况时，默认 **不改**：
- 页面已经显示正常
- 当前变量链路已经正确
- 当前 SEO 已由系统统一输出
- 当前页面结构与程序职责一致
- 改动只会“更像标准”，但没有真实收益

### 2.2 模板不能写死后台可变链接

凡是后台可改或程序可生成的链接，模板里都不能写死。例如：
- `fake_top`
- `fake_recentread`
- `fake_info_url`
- `fake_indexlist`
- 分类链接
- 搜索链接

### 2.3 SEO 优先服从程序实际输出链路

系统中已经存在：
- `shipsay/seo.php`
- `shipsay/configs/seo_tpl.php`

因此模板默认不能再为了“看起来完整”去硬写：

```html
<title>分站1</title>
<meta name="keywords" content="分站1">
<meta name="description" content="分站1">
```

正确规则：
1. 先复用系统现有 SEO 链路
2. 模板只负责输出已经准备好的 `seo_title / seo_keywords / seo_description`
3. 只有在程序未提供时，才允许模板给极轻的兜底值
4. 不允许模板层强行覆盖公共 SEO 逻辑

### 2.4 页面添加内容必须与页面气质一致

加模块不是越多越好。所有新增内容都必须满足：
- 与当前模板风格一致
- 不打断主阅读流
- 不造成空白异常或布局失衡
- 手机端能正常显示

## 5. 公共规则

## 5.1 头部与尾部

### Header 负责
- 站点 logo / 标题
- 搜索入口
- 首页 / 书库 / 完本 / 足迹 等公共入口
- 顶部导航

### Footer 负责
- 底部文案
- 版权区
- 公共统计脚本或站点尾部信息

### 规则
- 模板页统一使用 `__THEME_DIR__` include
- 静态资源统一走：`/static/<?=$theme_dir?>/`
- 搜索入口必须走程序/配置生成的可变链接，不能写死 `/search/`

## 5.2 手机端优先

验收时先看手机端，再看 PC 端。

必须满足：
- 不横向撑破
- 不错位
- 不出现大片无意义空白
- 不因加模块影响主体阅读/点击流
- 按钮可点，搜索可用，分页可用

## 5.3 标签一致性

页面中的：
- 标题
- 面包屑
- 按钮文案
- 模块标题
- SEO 标题

必须保持一致，不能页面写“月排行榜”，SEO 却还是“总排行榜”。

## 6. 各页面职责与标准

## 6.1 首页 `tpl_home.php`

### 必须保留
- 推荐书区
- 最近更新/最新入库之类的核心列表
- 顶部导航与搜索
- 站点基础入口

### 不再强制
- **不强制首页必须嵌入完整排行榜模块**

### v5.1 新规则
- 首页只要求提供 **排行榜入口** 即可
- 是否展示完整排行榜块，由模板风格决定
- 如果首页已经平衡，不要为了“模块齐全”硬塞排行块

## 6.2 栏目页 `tpl_category.php`

### 当前目标
- PC 端可分栏
- 手机端单列
- 长标题不撑坏布局

### 规则
- PC 端推荐双列或多列
- 长标题可限制字数 / 两行截断
- 简介区高度要有统一节奏
- 可加入长尾推荐链接，但不能抢主列表视觉

## 6.3 详情页 `tpl_info.php`

### 必须保留
- 封面 / 书名 / 作者 / 分类 / 状态 / 字数
- 最新章节信息
- 开始阅读 / 查看目录
- 作品简介
- 最新章节区
- 长尾相关推荐

### 明确禁止
- 为了“更简洁”把章节区删掉
- 为了“更统一”把相关推荐删掉

### 说明
详情页不是只有“书信息卡片 + 两个按钮”。

## 6.4 目录页 `tpl_indexlist.php`

### 必须保留
- 书籍基础信息
- 目录标题
- 开始阅读 / 返回详情
- 章节列表
- 分页逻辑（如启用分页）

### 规则
- 目录列表不能出现右半边大面积无意义留白
- 章节列表区宽度要贴合主内容
- 手机端目录按钮优先竖排或自然换行

## 6.5 阅读页 `tpl_reader.php`

### 必须保留
- 面包屑
- 章节标题
- 作者 / 字数 / 时间 等基础信息
- 正文输出：`<?=$rico_content?>`
- 上一章 / 下一章 / 返回目录 / 返回详情
- 阅读分页信息（如果启用阅读分页）

### 规则
- 正文区必须优先保证阅读体验
- 面包屑、分页、设置按钮不能压主阅读内容
- 不能因为“加功能”破坏正文可读性

## 6.6 搜索页 `tpl_search.php`

### 必须保留
- 搜索表单
- 结果列表
- 结果数量
- 搜索关键词标题一致性

### 规则
- 搜索表单样式要与头部保持一致
- 搜索结果卡片不能比首页/栏目页风格完全脱节

## 6.7 阅读记录 `tpl_recentread.php`

### v5.1 统一规则
- 单独一页，不与排行榜或推荐列表混用
- 头部入口固定保留
- 支持无封面占位
- PC / 手机端使用同一数据逻辑，不是两套含义不同的布局

## 6.8 排行榜 `tpl_rank.php / tpl_top.php / tpl_rank_list.php`

### 统一目标
- 聚合榜入口
- 单榜页
- 榜单切换
- 搜索区
- 手机端自适应

### 新规则
- 后台 `fake_top` 是唯一主入口前缀
- 单榜页链接也必须基于 `fake_top`
- 榜单切换必须自适应
- 不允许模板里继续写死 `/top/` 或 `/rank/`

---

## 7. v5.0 删减项（v5.1 明确收回）

以下条目在 v5.1 中明确弱化或删除：

1. **首页必须有完整排行榜模块** → 改为“首页必须有排行榜入口”
2. **模板默认补 SEO title/keywords/description** → 改为“优先服从系统 SEO 链路”
3. **为统一而删减详情页章节区/相关推荐** → 禁止
4. **只强调模板齐全，不强调页面职责** → 改为“职责优先”

---

## 8. 移动端验收标准

每一轮模板修改都必须过这一组检查：

### 首页 / 列表
- 图片、标题、简介、作者信息对齐正常
- 搜索框不压行、不偏移
- 导航不遮挡主内容

### 详情页
- 简介区无异常留白
- 按钮区位置自然
- 最新章节区可正常阅读

### 目录页
- 章节列表宽度正常
- 按钮与标题区不冲突
- 不留大面积空白

### 阅读页
- 面包屑不过长或挤压主内容
- 分页信息位置合理
- 设置按钮不抢正文阅读区

### 排行榜/搜索/足迹
- 手机端可点击
- 搜索框不变形
- 榜单切换可自适应
- 列表排版不溢出

---

## 9. 修改流程（AI / 人工共用）

每次改模板，按这个顺序：

1. 先确认当前源码包是最新基线
2. 先核变量来源
3. 先核页面职责
4. 先判断当前是否已经正确
5. 再决定改不改
6. 修改后先做基础语法检查
7. 再打包交付

不允许直接“看页面像有问题”就凭印象大改。

---

## 10. v5.1 第一轮建议范围

优先处理：
- `tpl_header.php`
- `tpl_home.php`
- `tpl_category.php`
- `tpl_info.php`
- `tpl_indexlist.php`
- `tpl_recentread.php`

第二批再处理：
- `tpl_reader.php`
- `tpl_rank.php`
- `tpl_top.php`
- `tpl_search.php`

原因：先收口结构与变量，再收口阅读细节与排行榜细节，冲突更少。

---

## 11. 一句话结论

**v5.1 的本质不是“加更多规则”，而是把规则收准：哪些必须保留、哪些不能写死、哪些不能乱删、哪些必须优先服从程序真实链路。**


## 9. 第一轮模板落地备注（2026-03-06）

### 已落地
- 首页移除强制完整排行榜块，改为顶部入口优先。
- 详情页恢复“最新章节”与“相关推荐（长尾入口）”。
- 目录页恢复为更适合 PC 的双列结构，并保留手机单列。
- 栏目页以“页面完整性优先”为准，开始收口标题长度与卡片节奏。
- 阅读记录页统一回归单独页面职责，不再和排行榜/推荐页混合。

### 本轮经验
- 不能因为追求统一，把已正确的章节区、相关推荐、SEO 调用链误删。
- 目录分页优先复用 app 已生成变量（如 `$htmltitle`），模板不要重复造逻辑。
- 同一文件连续微调容易反复冲突，相关问题应尽量合并为一轮交付。

## 10. v5.2 收口方向

1. 排行入口回归“顶部全站入口优先”，正文区不要重复塞入口。
2. 详情页必须同时保留：简介、最近章节、前 50 章预览、相关推荐。
3. 分类页分页必须服从程序真实总页数，禁止模板或 app 再额外写死 10 页上限。
4. 阅读页以“阅读流”优先：面包屑、分页信息、标题、正文左右对齐，手机端不得出现明显错位。
5. 用户可见的 HTML 注释、母模板说明文案一律删除。
6. 继续复核 `tpl_search.php / tpl_author.php / tpl_error.php` 与头部搜索链路的一致性。
